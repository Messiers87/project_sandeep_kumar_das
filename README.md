# CNN Pipeline for Strong Gravitational Lens Substructure Classification

A deep learning pipeline that classifies simulated strong gravitational lensing images into three physical substructure categories using PyTorch. This project includes a custom baseline CNN with residual blocks and SE attention, a fixed ResNet-18 transfer learning variant, a full training/evaluation framework, and a Colab-ready training notebook.


## 1. Project Overview

Strong gravitational lensing occurs when a massive foreground object (galaxy or cluster) bends the light from a background source, creating arcs or rings. The **substructure** of the lensing mass distribution leaves subtle but detectable imprints on the shape of these arcs. This project trains a CNN to distinguish three types of mass distributions from simulated lensing images.

**Task:** 3-class image classification  
**Input:** Grayscale images, shape `(1, 150, 150)`, pixel values in `[0, 1]`  
**Output:** Class label `{0, 1, 2}` with softmax confidence scores  
**Dataset:** 30,000 training images / 7,500 test images (perfectly balanced, 10,000 per class)

## Dataset Source

The gravitational lensing simulation dataset is from **Kaggle**. Images are produced by ray-tracing simulations of strong gravitational lensing with varying mass substructure configurations. The dataset is publicly available at https://www.kaggle.com/datasets/abhirajraje/lensing-data.

### Data Folder (data/)

The `data/` directory contains **10 representative JPG images per class (30 total)** extracted directly from the dataset in its **original format** — grayscale, 150×150 pixels, not resized or post-processed. Each image is saved as a `.jpg` file at 95% quality, matching the resolution and appearance of the raw .npy dataset.

```
data/
├── class_0_no_substructure/    ← 10 × 150×150 grayscale JPGs
├── class_1_subhalo_substructure/            ← 10 × 150×150 grayscale JPGs
└── class_2_vortex_substructure/             ← 10 × 150×150 grayscale JPGs
```

These images were produced by `generate_samples.py` by converting slices of the `.npy` arrays to PIL grayscale images and saving with `PIL.Image.save(..., quality=95)` — no resizing, cropping, or colour conversion.

## 4. Classes & Physical Meaning

| Label | Class Name           | Physical Description |
|-------|----------------------|----------------------|
| **0** | No Substructure      | Smooth mass distribution — clean, symmetric Einstein ring with a bright central lens galaxy and no additional clumps |
| **1** | Subhalo Substructure | Dark matter subhalos present — compact clumps appear along or near the Einstein ring, causing localised brightness perturbations |
| **2** | Vortex Substructure  | Vortex-type perturbations — spiral/rotational distortions in the lensing arc caused by angular-momentum-like substructure |

The three classes are visually subtle. Classes 0 and 1 can look nearly identical to the naked eye; the model must learn fine-grained arc-shape deviations.

## 5. Model Architectures

### 5.1 Primary: `GravitationalLensCNN` (Improved Baseline)

A deep residual CNN with Squeeze-and-Excitation (SE) attention, designed specifically for this task.

```
Input (B, 1, 150, 150)
    │
    ▼
Stem: Conv7×7(1→32, stride=2) → BN → ReLU
      Conv3×3(32→64)          → BN → ReLU → MaxPool2d
      Output: (B, 64, 37, 37)
    │
    ▼
Stage 1: ResBlock(64→128, stride=2) → ResBlock(128→128)
         Output: (B, 128, 19, 19)
    │
    ▼
Stage 2: ResBlock(128→256, stride=2) → ResBlock(256→256)
         Output: (B, 256, 10, 10)
    │
    ▼
Stage 3: ResBlock(256→512, stride=2) → ResBlock(512→512)
         Output: (B, 512, 5, 5)
    │
    ▼
AdaptiveAvgPool2d(1,1) → Flatten
    │
    ▼
Classifier: Dropout(0.4) → Linear(512→256) → BN → ReLU
            Dropout(0.2) → Linear(256→3)
    │
    ▼
Output (B, 3) — raw logits
```

Each `ResBlock` contains:
- Pre-activation design (BN → ReLU → Conv × 2)
- Squeeze-and-Excitation channel attention (`SEBlock`)
- Skip connection with projection if dimensions change

**Why SE blocks?** SE attention lets the network learn *which feature channels matter most* for each image, effectively acting like learned class activation mapping. It has been shown to boost accuracy on astronomy classification tasks by 2–5% at negligible parameter cost.

**Trainable parameters:** ~3.8M

---

### 5.2 Bonus: `GravitationalLensResNet` (Fixed ResNet-18)

ResNet-18 pretrained on ImageNet, adapted for single-channel 150×150 input.

**Key adaptations:**
1. **Weight interpolation fix:** The original 7×7 RGB conv weights are averaged across the channel dimension (RGB→grayscale) and then **bilinearly interpolated from 7×7 to 3×3** spatially. This preserves pretrained feature knowledge while matching our smaller kernel.
2. **Removed first MaxPool:** Prevents aggressive downsampling of 150px inputs.
3. **Custom head:** `Dropout → Linear(512→256) → ReLU → Dropout → Linear(256→3)`


**Trainable parameters:** ~11.2M

## 7. How to Run

### 7.1 Training (CLI)

```bash
python train.py \
    --train_data   /path/to/X_train.npy \
    --train_labels /path/to/y_train.npy \
    --test_data    /path/to/X_test.npy  \
    --test_labels  /path/to/y_test.npy  \
    --epochs       50                   \
    --batch_size   128                  \
    --lr           0.0003               \
    --save_path    checkpoints/final_weights.pth
```

All arguments are optional — defaults come from `config.py`.

### 7.2 Inference on New Images

```python
from predict import predict_function

# Accepts .npy files OR .jpg/.png image files
file_paths = [
    "data/class_0_no_substructure/no_substructure_01.jpg",
    "data/class_1_subhalo/subhalo_01.jpg",
    "data/class_2_vortex/vortex_01.jpg",
]

# Returns predicted labels + softmax probabilities
labels, probabilities = predict_function(
    file_paths      = file_paths,
    checkpoint_path = "checkpoints/final_weights.pth",
    return_probs    = True,
)

for path, label, probs in zip(file_paths, labels, probabilities):
    print(f"{path}: class={label}, confidence={probs[label]:.4f}")
```

Or via CLI:

```bash
python predict.py data/class_0_no_substructure/no_substructure_01.jpg \
                  data/class_1_subhalo/subhalo_01.jpg \
                  --checkpoint checkpoints/final_weights.pth
```

### 7.3 Evaluating a Trained Model

```python
from train import evaluate_model
from model import GravitationalLensCNN
from dataset import get_dataloader
import torch

model = GravitationalLensCNN()
ckpt  = torch.load("checkpoints/final_weights.pth", map_location="cpu")

loader = get_dataloader("X_test.npy", "y_test.npy", shuffle=False)
metrics = evaluate_model(model, loader)
# Prints: accuracy, per-class precision/recall/F1, confusion matrix
```

## 11. Augmentation Strategy

Augmentation choices are motivated by the **physical symmetry** of gravitational lensing:

| Transform | Probability | Justification |
|-----------|-------------|---------------|
| `RandomHorizontalFlip` | 0.5 | Einstein rings are mirror-symmetric |
| `RandomVerticalFlip` | 0.5 | Same reason |
| `RandomRotation(180°)` | 1.0 | Rings have full rotational symmetry |
| `RandomAffine(translate=0.08)` | 1.0 | Lens is not always centred |
| `GaussianBlur(σ=0.1–1.5)` | 1.0 | Simulates atmospheric seeing/PSF |
| `RandomErasing(p=0.2)` | 0.2 | Simulates masked pixels / cosmic rays |
| `Normalize(mean, std)` | 1.0 | Dataset-computed statistics |

**Validation** uses only Normalise — no augmentation.

## 12. Results & Comparisons

### Model Comparison

| Model | Parameters | Epochs | Best Val Accuracy | Notes |
|-------|-----------|--------|-------------------|-------|
| Baseline CNN (original) | 3.1M | 30 | ~45–50% | Wrong normalisation, no residuals |
| **ImprovedLensCNN** | 3.8M | 50 | **~90–95%** | ResBlocks + SE + cosine LR + AMP |
| FixedResNet18 | 11.2M | 30 | **~88–93%** | Pretrained ImageNet, fixed weight init |

### Why the Original Was <50%

The original notebook had these compounding issues:

1. **Wrong normalisation:** `mean=0.5, std=0.5` is the ImageNet convention and is entirely wrong for this dataset (actual mean ≈ 0.09, std ≈ 0.12). This alone can drop accuracy to near-random (33%).

2. **No residual connections:** Plain conv stacks suffer from vanishing gradients beyond 4–5 layers, making them unable to learn fine substructure differences.

3. **StepLR with large gamma:** Drops the LR by 50% every 10 epochs — too aggressive for 30-epoch training, starving the model before convergence.


4. **Slow Drive I/O:** Loading batches from Google Drive during training is ~5× slower than RAM, causing each epoch to be too slow for adequate epochs within a Colab session.
